Types of Linked Lists in Programming Language.

NKWASHI SHINGA,
Technical Writer and Editor


Linked list is a linear structure consisting of a sequence of elements where each element points to the next element in the sequence. Each element also known as a node contains both the data and a reference(or link) to the next node in the sequence. The last node in the list points to null, indicating the end of the list.
Linked lists provides dynamic memory allocation, efficient insertion and deletaion of elements, and flexibility in managing memory. However, they have slower access times compared to arrays since elements are not stored in contiguous memory locations.

There are three types of linked lists which are common in python, which are:

  1. Single Linked List:
    In a single linked list each node has a reference only to the next node in the sequence.
    It contains two components: the data and the next pointer. The first node is called the head, and the last node ponts to null.

  2. Double Linked List:
    In a double linked list, each node has references to both the previous and the next nodes in the sequences. It allows traversal in both directions. Besides the data and next pointer, each node also has a previous pointer.

  3. Circular Linked List:
    In a circular linked list, the last node points back to the first node instead of null, creating a circular structure. It can be implemented as either a single or double linked list. circular linked lists are useful in scenarios where contionuous traversal is required.

    4. Linked lists offer several advantages and use cases:

    1. Dynamic Size:
      Unlike arrays, linked lists can dynamically grow and shrink as elements are added or removed. This flexibility makes linked lists useful in situations where the size of the data may change over time.

    2. Efficient Insertion and Deletion:
      Insertion and deletion operations can be performed efficiently in a linked list by updating the references between nodes. In contrast, arrays may require shifting elements, resulting in slower performance.

    3. Memory Efficiency:
      Linked lists can be more memory-efficient than arrays since they only use memory proportional to the number of elements in the list. Arrays, on the other hand, may require additional space to accommodate potential growth.

    4. Implementation of other Data Structures: Linked lists serve as building blocks for other data structures such as stacks, queues, and graphs. For example, a stack can be implemented using a singly linked list where the top of the stack is the head of the list.

    However, linked lists also have some disadvantages:

    1. Random Access:
      Unlike arrays, accessing an element in a linked list requires traversing the list from the beginning, which can be slower if direct access to elements is required frequently.

    2. Extra Memory Overhead:
      Linked lists require additional memory to store the references or pointers between nodes. This overhead can be significant when storing small amounts of data.

    Overall, linked lists are a fundamental data structure that offers flexibility, efficient insertions and deletions, and is widely used in various algorithms and applications.
